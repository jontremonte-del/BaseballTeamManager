🔥 Firebase Setup Instructions

Follow these steps once to enable cloud sync so your coaches can see the lineup on their devices.

1
Create a free Firebase project
Go to console.firebase.google.com, click Add project, name it diamond-manager, disable Google Analytics (optional), click Create project.
2
Add a Web App
In your project, click the </> web icon, give it a nickname like diamond-manager-web, click Register app. You'll see a firebaseConfig object — copy the whole thing.
3
Enable Realtime Database
In the left sidebar go to Build → Realtime Database, click Create database, choose Start in test mode, click Enable. Copy the database URL (looks like https://your-project-default-rtdb.firebaseio.com).
4
Paste config into the code
Open index.html in GitHub. Find the FIREBASE_CONFIG object near the top (inside the <script type="module"> block) and replace each PASTE_YOUR_..._HERE value with the values from your Firebase config. Make sure databaseURL is included — paste it from step 3 if it's missing.
5
Commit & deploy
Save the file in GitHub (Commit changes). Vercel redeploys automatically in ~30 seconds. Reload the app and the sync bar at the top should turn green.
6
Share with coaches
Send coaches your Vercel URL (e.g. baseball-team-manager-roan.vercel.app). They open it in any browser — all data syncs in real time. No login needed.
⚠️ Test mode database rules expire after 30 days. Before expiry, go to Realtime Database → Rules and set "read": true, "write": true to keep it open, or add authentication for more security.
